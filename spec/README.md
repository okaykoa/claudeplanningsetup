# Planning

These documents define the project in its entirety. When complete, they constitute a spec from which an engineer can implement the project. Fill them out in order: principles → design → requirements → roadmap. In any agent configuration files (e.g. `CLAUDE.md`), point to this directory as a context source.

| Doc | Purpose | Authority |
|---|---|---|
| `principles.md` | Guiding values and philosophy behind the project | Highest |
| `design.md` | The functionality, data, and model that define the product | Second |
| `requirements.md` | Technical requirements and implementation decisions | Third |
| `roadmap.md` | Milestones, steps, and definitions of done | Lowest |

When docs conflict, the higher-ranked doc takes precedence. Never resolve a conflict without informing the user and asking first.
